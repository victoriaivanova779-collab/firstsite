Grundprinzip von HTML

HTML liefert die Struktur, nicht das Design und nicht die Logik. Der Browser liest die Auszeichnung, baut daraus ein DOM und rendert dieses.

Mit ein paar wenigen Tags lassen sich die meisten Inhalte darstellen. HTML ist stabil, universell und zukunftssicher.

Mehr über das DOM erfahren

Das DOM ist eine Baumstruktur, die alle HTML-Elemente repräsentiert. JavaScript kann darauf zugreifen und Elemente dynamisch ändern.

Textauszeichnung

Fett (strong) für wichtige Betonung.
Fett (b) für stilistische Hervorhebung ohne besondere Bedeutung.
Kursiv (em) für betonte Aussprache.
Kursiv (i) für technische Begriffe oder Fremdwörter.
Markiert (mark) für hervorgehobene Textpassagen.
Klein (small) für Kleingedrucktes oder Nebensätze.
Durchgestrichen (del) für gelöschten Text.
Unterstrichen (ins) für eingefügten Text.
Text mit Tiefstellung (sub) wie in H2O.
Text mit Hochstellung (sup) wie in x2.
Code (code) für inline Code-Snippets.
Tastatur (kbd) wie Strg + C.
Sample Output (samp) für Programmausgaben.
Variable (var) für mathematische Variablen wie x.

Übersicht semantischer Elemente

  • <header> – Kopfbereich einer Seite oder eines Abschnitts
  • <nav> – Navigationsbereich
  • <main> – Hauptinhalt der Seite
  • <section> – Thematischer Abschnitt
  • <article> – Eigenständiger Inhalt (z. B. Blogpost)
  • <aside> – Ergänzender Inhalt (z. B. Sidebar)
  • <footer> – Fußzeile
  • <figure> und <figcaption> – Abbildung mit Beschriftung
  • <time> – Zeitangabe
  • <address> – Kontaktinformationen
